2 Konsep Dasar Virtualisasi & CloudSim Plus

2.1 Apa itu Virtualisasi?

Virtualisasi adalah teknologi yang memungkinkan satu server fisik (hardware) menjalankan beberapa sistem operasi atau aplikasi secara bersamaan melalui Virtual Machine (VM). Teknologi ini menjadi fondasi utama cloud computing.

💡 Analogi Sederhana: Bayangkan sebuah apartemen (server fisik) yang dibagi menjadi beberapa unit kamar (VM). Setiap penghuni (cloudlet/aplikasi) mendapat kamar sendiri yang terisolasi, tapi mereka berbagi infrastruktur gedung yang sama (listrik, air, internet).

2.2 Tipe Hypervisor

TipeNamaPosisiContoh
Type 1Bare-metalLangsung di atas hardwareVMware ESXi, Xen, KVM
Type 2HostedDi atas OS hostVirtualBox, VMware Workstation

2.3 Komponen CloudSim Plus

KomponenRepresentasiFungsi
CloudSimPlusSimulation EngineMengatur jalannya simulasi dan event
DatacenterBrokerPelanggan CloudPerantara yang mengirim VM & cloudlet ke datacenter
DatacenterPusat DataMenyediakan resource komputasi
HostServer FisikMemiliki CPU, RAM, BW, Storage
PECPU CoreProcessing Element, diukur dalam MIPS
VMVirtual MachineMesin virtual yang berjalan di atas host
CloudletPekerjaan/AplikasiTask yang dijalankan pada VM

2.4 Alur Simulasi CloudSim Plus

Alur: CloudSimPlus → Broker submit VM & Cloudlet → Datacenter menerima → Host mengalokasikan VM → VM menjalankan Cloudlet → Hasil ditampilkan sebagai tabel.

3.1 Source Code

var simulation = new CloudSimPlus(); var broker = new DatacenterBrokerSimple(simulation); // Host: RAM 10 GB, Strg 100 GB, BW 100 Gbps, 1 PE @ 20000 MIPS var host = new HostSimple(10000, 100000, 100000, List.of(new PeSimple(20000))); new DatacenterSimple(simulation, List.of(host)); // VM: 1000 MIPS, 1 PE var vm = new VmSimple(1000, 1); vm.setRam(1000).setBw(1000).setSize(1000); // Cloudlet: 10000 MI, 1 PE, utilisasi 50% var utilization = new UtilizationModelDynamic(0.5); var cloudlet1 = new CloudletSimple(10000, 1, utilization); var cloudlet2 = new CloudletSimple(10000, 1, utilization);

8.2 Perbandingan TimeShared vs SpaceShared

⏱️ TimeShared (Latihan 5)

  • Cloudlet 0: Start 0.1s, Finish 20.1s
  • Cloudlet 1: Start 0.1s, Finish 20.1s
  • CPU per cloudlet: 500 MIPS
  • Keduanya jalan bersamaan

📦 SpaceShared (Latihan 6)

  • Cloudlet 0: Start 0.1s, Finish 10.1s
  • Cloudlet 1: Start 10.2s, Finish 20.2s
  • CPU per cloudlet: 1000 MIPS (penuh)
  • Cloudlet 1 menunggu Cloudlet 0
🔍 Kesimpulan: SpaceShared memberi tiap cloudlet akses CPU penuh (ExecTime 10s vs 20s), tapi cloudlet berikutnya harus mengantri. TimeShared memungkinkan semua jalan bersamaan tapi waktu individu lebih lama. Total throughput keduanya sama (~20 detik).

14.3 Poin Penting

  1. Virtualisasi memungkinkan efisiensi penggunaan hardware melalui pembagian resource ke banyak VM.
  2. TimeShared cocok untuk interactive workloads (semua jalan bersamaan), sedangkan SpaceShared cocok untuk batch processing (satu per satu, tapi lebih cepat per task).
  3. Multi-tenancy memungkinkan beberapa pelanggan cloud berbagi infrastruktur yang sama dengan isolasi yang terjamin.
  4. Horizontal Scaling lebih diutamakan di cloud computing karena fault-tolerant dan elastis.
  5. CloudSim Plus adalah alat yang efektif untuk mempelajari konsep cloud computing melalui simulasi tanpa memerlukan infrastruktur nyata.
